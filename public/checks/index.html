Check register

Every requirement the suite checks, cited to the spec

The spck.dev conformance suite is built from a requirements register scraped from the UCP specification and quote-verified against SHA-pinned sources. This page lists the 318 requirements the suite currently covers; each links to the requirement text and its exact spec citation. For the full picture including requirements not yet covered, see the coverage matrix; for how verdicts are graded, the rubric.

Signatures 23

  • SIG-001UCP signs and verifies HTTP messages using RFC 9421 HTTP Message Signatures across all HTTP-based transports.
  • SIG-002All implementations MUST support verifying P-256 (ES256) signatures.
  • SIG-006The algorithm is derived from the key's crv field in the JWK; alg is NOT included in Signature-Input parameters.
  • SIG-007Public keys MUST be represented using JSON Web Key (JWK) format as defined in RFC 7517.
  • SIG-008Public keys are published in the signing_keys array of the party's UCP profile at /.well-known/ucp.
  • SIG-010Content-Digest follows RFC 9530 and hashes the raw body bytes; implementations MUST use sha-256.
  • SIG-012Content-Digest header is required when a request/response has a body.
  • SIG-013Signature-Input and Signature headers are required on requests and responses.
  • SIG-014REST request signed components MUST include @method, @authority, and @path (all Required).
  • SIG-015REST request signed components MUST include content-digest and content-type when the request has a body.
  • SIG-017@query MUST be a signed component if the request has query parameters.
  • SIG-019ECDSA signatures MUST use fixed-width raw r||s encoding per RFC 9421, not ASN.1/DER (64 bytes P-256, 96 bytes P-384).
  • SIG-020REST response signed components use @status instead of @method (Required).
  • SIG-021REST response signed components MUST include content-digest and content-type if the response has a body.
  • SIG-025On duplicate idempotency key, return cached response and do not re-execute.
  • SIG-027Webhook notifications MUST be signed.
  • SIG-031signature_missing maps to HTTP 401 (required signature header/field not present).
  • SIG-032signature_invalid maps to HTTP 401 (signature verification failed).
  • SIG-033key_not_found maps to HTTP 401 (key ID not found in signer's signing_keys).
  • SIG-034digest_mismatch maps to HTTP 400 (body digest doesn't match Content-Digest header).
  • SIG-036Verification must reconstruct the signature base and reject with signature_invalid if ECDSA verification fails.
  • SIG-037Verification must return key_not_found when the kid is not present in the signer's signing_keys.
  • SIG-038Verification must return digest_mismatch when Content-Digest does not match the recomputed sha-256 of the body bytes.
